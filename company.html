雲端服務三巨頭

2015年 ─ 物聯網進入戰國時代


雲端的技術可使企業往後導入IoT變得更容易，

也能帶來更多客製化IoT平臺新選擇。


  • Google
    ( IoT OS - Brillo )

    以Android打造的開源IoT作業系統Brillo，讓Android也能使用IoT，更可在同個網路下的裝置間，具有相互辨識和溝通能力。

  • AWS 亞馬遜
    ( AWS IoT )

    AWS提供了從裝置端、閘道端一路到雲端的IoT租用服務，可讓企業利用IoT更為簡單；同時也提供規則引擎來簡化處理IoT裝置的資訊。

  • Microsoft 微軟
    ( Auzre IoT )

    Azure打造一個多租用戶雲端服務的IoT PaaS平臺，並先做為遠端監控、預測維護和資產管理等企業IoT應用。

如何建立物聯網？

架構、環境、平臺、裝置，缺一不可


Intel 參戰

Intel在2015年11月，發表了結合軟、硬體的 IoT 平臺參考架構，誓言要建立一個通吃 IoT 裝置、網路和雲端的生態系，並推出一個可信賴分析平臺，能結合多種開源資料分析工具，來提供各類 IoT 分析應用，而在這3大雲端巨頭和其他網路、IT廠商紛紛搶占 IoT 平臺之下，也讓 2016 年的 IoT 平臺大戰更加白熱化。


5G 技術崛起

物聯網最重要的關鍵就在於「網路」，裝置上的資料必須經由網路媒介來進行回傳，然而 5G 技術的推動有助於物聯網平台和服務，其中 IoT 強調低功耗、廣域連接等特色，而 5G 的低延遲時間、廣域覆蓋率、高通訊速率就能支援 IoT 部署所需的關鍵要素。

隨著有線與無線技術規格持續翻新，未來也將能支援更高速連網及更低功耗的 Always On 運作模式，期望可以帶動更多 IoT 應用成形。
